# Create Base Standards

Debes comprobar si existe el documento openspec/specs/base_standards.md y si no crearlo con la información: Este documento define un conjunto de estándares, lineamientos y prácticas recomendadas para el desarrollo de un proyecto en la capa de servidor. Especifica cómo estructurar el código, cómo organizar los componentes del sistema y cómo aplicar principios de diseño para lograr una base de código coherente, mantenible y escalable.

Incluye orientaciones sobre:

Arquitectura general del proyecto, describiendo cómo deben separarse las responsabilidades y cómo deben interactuar las distintas capas.

Modelado del dominio, con pautas sobre la definición de entidades, objetos sin identidad, agrupaciones lógicas y componentes encargados de gestionar operaciones del dominio.

Principios de diseño, como responsabilidad única, separación de interfaces, inversión de dependencias, reutilización y extensibilidad.

Normas de codificación, incluyendo convenciones de nombres, validación, manejo de errores, uso de tipos y pautas para registros de auditoría o depuración.

Diseño de interfaces externas, presentando estándares para definir puntos de entrada, formatos de solicitud y respuesta, manejo de errores y configuración transversal.

Patrones para interacción con fuentes de datos, abarcando definición de esquemas, control de versiones, acceso a datos y abstracciones para repositorios.

Criterios y prácticas de pruebas, que incluyen organización de archivos, convenciones de nomenclatura, estructura de casos, uso de dobles de prueba, cobertura mínima y validación de flujos felices, errores y casos límite.

Buenas prácticas de rendimiento y seguridad, con pautas para optimizar operaciones, gestionar datos sensibles y validar entradas.

Flujo de trabajo de desarrollo, cubriendo control de versiones, automatización, validaciones previas a integrar cambios y estándares de calidad.

Consideraciones para despliegue, incluyendo lineamientos sobre empaquetado, configuración del entorno y ejecución en plataformas administradas.

En esencia, este documento funciona como una guía integral de normas de ingeniería, destinada a asegurar uniformidad, claridad técnica y robustez en todo el ciclo de desarrollo del proyecto.
