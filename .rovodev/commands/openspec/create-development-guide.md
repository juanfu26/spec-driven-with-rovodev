# Create Development Guide

Debes comprobar si existe el documento openspec/specs/development_guide.md y si no crearlo con la información: Este documento ofrece una guía de desarrollo que describe, de manera secuencial, cómo preparar el entorno necesario para trabajar en un proyecto, así como la forma de ejecutar sus distintas suites de pruebas.

Incluye instrucciones para:

Verificar requisitos previos y herramientas necesarias antes de comenzar.

Obtener el código fuente desde su repositorio y ubicarse en la estructura del proyecto.

Configurar los parámetros de entorno, diferenciando los necesarios para distintos componentes del sistema.

Iniciar y preparar los servicios auxiliares requeridos por la aplicación, incluyendo la puesta en marcha de sus dependencias externas.

Instalar y configurar cada parte del proyecto, tanto en la capa de servidor como en la capa de cliente, incluyendo la generación de artefactos internos y la inicialización de datos.

Ejecutar la aplicación en modo de desarrollo, indicando dónde quedan accesibles sus distintos módulos.

Configurar y utilizar las herramientas de pruebas, tanto para pruebas automatizadas de servidor como para pruebas unitarias y de experiencia de usuario en la capa de cliente.

Ejecutar los distintos tipos de pruebas disponibles, así como opciones adicionales como ejecución interactiva o ejecución con seguimiento de cobertura.

En resumen, el documento funciona como una hoja de ruta práctica que guía a cualquier desarrollador a preparar el entorno, iniciar el proyecto y ejecutar sus pruebas de forma correcta y consistente.
