# P1 – Recopilación de Requisitos (Plantilla Operativa)

Objetivo
- Recopilar requisitos de múltiples partes interesadas (clientes, expertos internos/externos y directivos) para producir un Documento de Especificaciones de Requisitos del Software (SRS) que establezca objetivos comunes, permita estimar costes, definir planificación y disponer de un plan detallado para alcanzar los objetivos.

Contexto de uso
- Descripción breve del producto/proyecto
- Stakeholders: <clientes | negocio | soporte | ventas | legal | seguridad | arquitectura | operaciones>
- Restricciones conocidas: <tiempos, presupuesto, compliance, plataformas>
- Referencias: wiki/ (o $WIKI_ROOT), tickets, documentos previos

Instrucciones
1) Reúne información desde todas las partes interesadas. Si existen materiales en wiki/, léelos y cítalos.
2) Usa sequential-thinking para organizar el análisis: objetivos → alcance → requisitos → riesgos → estimaciones → planificación.
3) Clasifica requisitos en funcionales y no funcionales (rendimiento, seguridad, disponibilidad, cumplimiento, mantenibilidad, observabilidad, etc.).
4) Define criterios de aceptación de alto nivel y dependencias clave.
5) Propón estimaciones iniciales (tamaño/orden de magnitud) y supuestos.
6) Esboza un plan y cronograma de alto nivel (hitos) para la siguiente fase.
7) No realices cambios en Jira/Confluence; sólo análisis y redacción del SRS.

Entrada esperada (rellenar al usar)
- Producto/área: <texto>
- Objetivos de negocio: <lista>
- Principales usuarios/personas: <lista>
- Fuentes disponibles en wiki/: <paths>

Salida esperada (Documento SRS resumido)
- Resumen ejecutivo (3–5 bullets)
- Objetivos y métricas de éxito (KPI)
- Alcance y exclusiones (out of scope)
- Requisitos funcionales (epics/capacidades)
- Requisitos no funcionales (seguridad, performance, disponibilidad, etc.)
- Criterios de aceptación de alto nivel
- Dependencias y supuestos
- Riesgos y mitigaciones iniciales
- Estimación inicial (tamaño/orden de magnitud) y costes aproximados
- Plan y cronograma de alto nivel (hitos)
- Referencias (paths en wiki/ y otros docs)

Notas
- El texto base: "El equipo de desarrollo recopila requisitos de varias partes interesadas, como clientes, expertos internos y externos, así como directivos, para crear un documento de especificaciones con los requisitos del software. El documento establece las especificaciones y define los objetivos comunes que ayudan a planificar el proyecto. El equipo estima los costos, define una programación y dispone de un plan detallado para conseguir los objetivos." debe orientar el tono y resultado.
